## The root cause of the negative “Actual” performance

The current performance-chart route is doing this conceptually:

### It sets an anchor date:

* **Jan 1, 2026**

### Then it calculates opening value using:

* **today’s fiat wallet balances**
* **today’s crypto wallet balances**
* **today’s stablecoin balances**
* **all investment records**, including ones created after Jan 1, 2026

That means the first point is **not a true historical opening value**.

So even if your investments are up, the graph can still show “Actual” performance as negative because the starting line is inflated or mis-timed.

## 3) Make the historical line use the same source as the opening point

Right now:

* opening point = manually reconstructed
* later points = snapshots

That is inconsistent.

Use this instead:

```ts
const snapshots = await storage.getPortfolioSnapshots(userId, ANCHOR, today);
const sortedSnapshots = [...snapshots].sort(
  (a: any, b: any) => new Date(a.snapshotDate).getTime() - new Date(b.snapshotDate).getTime()
);

const historyByMonth = new Map<string, { value: number; source: string }>();

for (const s of sortedSnapshots) {
  const d = new Date(s.snapshotDate);
  const key = `${d.getFullYear()}-${String(d.getMonth() + 1).padStart(2, '0')}`;
  historyByMonth.set(key, {
    value: Math.round(parseFloat(s.totalValue)),
    source: (s as any).source ?? "actual",
  });
}

const openingValue =
  sortedSnapshots.length > 0 ? Math.round(parseFloat(sortedSnapshots[0].totalValue)) : 0;
```

Then use `openingValue` from snapshots only.

# Bottom line

## The main bug is not the investment formula now.

The bigger bug is the **historical anchor basis**.

Your current model is:

* investment page records are linked
* return formulas are mostly corrected
* investment performance endpoint is no longer broken

But the **portfolio performance graph** is still wrong because:

1. it builds the opening point from **current wallet balances**
2. it includes **future investments** in the opening investment value
3. it mixes **manual opening reconstruction** with **snapshot-based later history**

That is why “Actual” can appear negative even when the investment book itself is positive.

## The correct model should be:

* **Investment page** creates/updates investment records
* **Shared valuation engine** calculates product-level returns
* **Snapshots** provide historical portfolio points
* **Performance chart** uses snapshots for all historical points
* **Forecast** starts from the last historical point

That is also consistent with the earlier identified requirement for one shared valuation basis across all pages.
