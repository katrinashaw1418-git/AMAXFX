You’re very close — this is now a **high-quality compliance UX**, but you need one key design principle:

👉 **Separate “User Editable Data” from “Verified Data (locked)”**

Right now everything looks editable → that weakens the credibility of verification.

---

# 🎯 CORE DESIGN MODEL (what you should implement)

## 🧠 Split into 3 layers

### 1. ✏️ User Input (editable)

### 2. 🔒 Verified Data (locked)

### 3. 🏢 Verification Source (Sumsub / external)

---

# ✅ HOW TO DESIGN IT (exact UI structure)

---

## 🔹 SECTION 1 — PERSONAL INFORMATION

👉 Keep editable

```text
Full Legal Name        John Chen          ✏️ Edit
Date of Birth          09/04/1988         ✏️ Edit
Nationality            Australia          ✏️ Edit
Mobile Number          0412345678         ✏️ Edit
```

---

## 🔹 SECTION 2 — RESIDENTIAL ADDRESS

👉 Editable BUT with verification state

```text
Residential Address

LEVEL 2, KINGS COURT
SYDNEY NSW 2216
Australia

Status: 🔴 Not Verified
[Upload document]
```

---

### AFTER verification:

```text
Status: 🟢 Verified via external provider
🔒 This address has been verified and cannot be edited
```

---

## 🔹 SECTION 3 — IDENTITY VERIFICATION (CRITICAL)

👉 This should NEVER be editable

---

### BEFORE:

```text
Status: In Progress
Provider: Sumsub

[Start Verification]
```

---

### AFTER:

```text
Status: 🟢 Verified
Verified via: Sumsub (external provider)

Verified Name: John Chen
Verified DOB: 09/04/1988

🔒 Verified information cannot be edited
```

---

# 🚨 IMPORTANT RULE

👉 Once verified:

* ❌ Remove Edit button
* ❌ Do NOT allow overwrite
* ❌ Do NOT let user “change identity”

# 🚀 FINAL STRUCTURE (CLEAN VERSION)

## 🧾 Identity & Compliance

### Identity Verification

* Status: 🟢 Verified
* Provider: Sumsub
* 🔒 Locked

---

### Residential Address

* Status: 🟡 Pending / 🟢 Verified
* Editable → until verified
* Locked after verification

---

### Personal Information

* Editable (non-regulated fields only)

---

# ⚠️ BIGGEST MISTAKE TO AVOID

❌ Letting users edit verified data
❌ Not showing who verified it
❌ Mixing editable + verified in same UI

---

# 🧠 ONE-LINE RULE

👉 **If it’s verified — it must be locked and attributed**
