# 2. 🗄️ Database Schema (Production-Grade)

## 2.1 Users

```sql
users (
  id UUID PK,
  email TEXT UNIQUE,
  phone TEXT,
  kyc_status TEXT,
  created_at TIMESTAMP
)
```

---

## 2.2 Wallets (per user, per asset)

```sql
wallets (
  id UUID PK,
  user_id UUID FK,
  asset TEXT, -- AUD, BTC, ETH, etc.
  balance DECIMAL(36,18) DEFAULT 0,
  locked_balance DECIMAL(36,18) DEFAULT 0,
  created_at TIMESTAMP
)
```

---

## 2.3 Transactions (master record)

```sql
transactions (
  id UUID PK,
  user_id UUID,
  type TEXT, -- DEPOSIT | WITHDRAWAL | TRANSFER | CONVERSION
  rail TEXT, -- BANK | PAYID | CARD | CRYPTO
  asset TEXT,
  amount DECIMAL(36,18),
  status TEXT, -- CREATED | PENDING | PROCESSING | COMPLETED | FAILED
  reference_code TEXT,
  metadata JSONB,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
)
```

---

## 2.4 Ledger Entries (double-entry accounting)

```sql
ledger_entries (
  id UUID PK,
  transaction_id UUID FK,
  wallet_id UUID FK,
  direction TEXT, -- DEBIT | CREDIT
  amount DECIMAL(36,18),
  created_at TIMESTAMP

```

# 3. 🔁 Transaction State Machine

All flows must follow:

```
CREATED
  ↓
PENDING (awaiting user action / external funds)
  ↓
PROCESSING (admin/system validating)
  ↓
COMPLETED
  ↓
FAILED
```

### Transitions:

* Deposit:

  * CREATED → PENDING → PROCESSING → COMPLETED
* Withdrawal:

  * CREATED → PENDING REVIEW → PROCESSING → COMPLETED/FAILED

# 6. 💰 Deposit Flows (Fixed UX + Backend)

## 6.1 Bank Transfer Deposit

### Backend:

1. User initiates deposit
2. System generates:

   * reference_code (unique)
   * transaction record (PENDING)
3. Store deposit instructions

### UI MUST show:

* Bank name
* BSB
* Account number
* Account name
* **Reference code (mandatory)**

### User action:

* Transfers funds externally

### Reconciliation:

* Admin/system matches incoming transfer using:

  * reference_code
  * amount
  * timestamp

---

## 6.2 PayID Deposit

### UI MUST show:

* PayID identifier (email/phone)
* Copy button
* Instructions:

  * “Send from your bank using PayID”
  * “Include reference code”

### Backend:

* Transaction created (PENDING)
* Matching via PayID metadata + reconciliation rules

---

## 6.3 Card Deposit via Stripe

Using Stripe:

* Redirect to hosted checkout
* Webhook confirms success
* Update transaction:

  * PENDING → COMPLETED
* Credit fiat wallet

---

# 7. 💸 Withdrawal Flows (Fixed UX + Backend)

## 7.1 Bank Withdrawal

### UI form must include:

* Account name
* BSB
* Account number
* Amount
* Confirmation step

### Backend:

1. Validate input
2. Check balance
3. Create transaction (PENDING REVIEW)
4. Lock funds (locked_balance++)
5. Await admin/system approval

---

## 7.2 PayID Withdrawal

### UI:

* Email/phone input
* Confirmation screen

### Backend:

* Same as bank withdrawal but method = PAYID

---

## 7.3 Crypto Withdrawals (Manual ops)

* Stored as request
* Status = PENDING REVIEW
* Ops team executes externally

---

# 8. 🔄 Internal Crypto Transfer (Fixed)

### Flow:

1. Input recipient email
2. Resolve userId
3. Validate wallet balance
4. Execute atomic ledger entries:

* Debit sender wallet
* Credit recipient wallet

### Must be:

* Atomic DB transaction
* Idempotent
* Logged in ledger_entries
