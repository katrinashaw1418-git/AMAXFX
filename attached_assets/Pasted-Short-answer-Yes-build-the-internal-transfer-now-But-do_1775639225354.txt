# 🏗️ What “Correct” Internal Transfer Looks Like

## 🔁 Flow (must follow this exactly)

```text
Sender submits → backend validates → atomic ledger move → transaction records → audit log
```

---

## 🔐 1. Identity Resolution (CRITICAL)

```ts
email → userId → wallet
```

### Must enforce:

* Exact match (case insensitive normalized)
* Verified user only (KYC level ideally)
* Cannot transfer to yourself (or handle explicitly)

---

## 💰 2. Atomic Ledger Movement (NON-NEGOTIABLE)

Must be inside **ONE DB transaction**

```ts
db.transaction(async (tx) => {
  debit sender
  credit receiver
  insert sender transaction
  insert receiver transaction
})
```

👉 If one fails → ALL fails

---

## 📒 3. Dual Transaction Records

You must create **two records**:

### Sender

```ts
type: "transfer_out"
direction: "out"
```

### Receiver

```ts
type: "transfer_in"
direction: "in"
```

👉 This is critical for:

* audit
* statements
* compliance

---

## 🧾 4. Audit Trail (VERY IMPORTANT)

Store:

* senderId
* receiverId
* timestamp
* IP (optional)
* reference ID (shared between both sides)
