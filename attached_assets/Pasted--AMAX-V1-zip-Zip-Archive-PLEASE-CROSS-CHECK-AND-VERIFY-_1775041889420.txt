Main errors and what to fix
1) Severe bug in /api/portfolio/allocation
The investment calculation is called with the wrong argument order.

Current logic:

calculateInvestmentPerformance(investment, product, evaluationDate)

But the function expects:

calculateInvestmentPerformance(product, investedAmount, investmentDate, currentDate)

Why this is a problem
This will produce invalid math for investment values, and can lead to NaN, broken percentages, broken allocation totals, and misleading charts/cards wherever allocation is used.

Fix
Replace that call with:

const investedAmount = parseFloat(investment.investedAmount);
const investmentDate = new Date(investment.investmentDate);
const performance = calculateInvestmentPerformance(product, investedAmount, investmentDate, evaluationDate);
2) Fiat allocation mixes currencies without conversion
In /api/portfolio/allocation, fiat balances are summed like this:

USD + CAD + EUR + GBP + AUD + HKD + SGD

That is not common-sense math. Those balances must be converted to a common currency first.

Why this is a problem
Your /api/portfolio endpoint correctly converts fiat balances to USD equivalent, but /api/portfolio/allocation does not. So the same portfolio can show different totals depending on the screen.

Example
Using the seeded wallet balances:

Raw fiat sum = 627,517

USD-equivalent fiat sum is only about 343,158

Fix
Use the same FX-conversion logic from /api/portfolio inside /api/portfolio/allocation.

3) “IRR” is being used incorrectly
The code labels returns as “IRR” or “annual return,” but calculates them as:

performanceFactor = 1 + (annualReturn * timeProgress)
That is simple linear growth, not IRR-style compounding.

Why this matters
If you say 11% IRR, users expect compounding logic, not simple prorating.

Fix
Use:

performanceFactor = Math.pow(1 + annualReturn, timeProgress)
If you want to keep simple prorating, then rename the metric to something like:

“simple annualized return assumption”

not IRR
