The correct model for a non-custodial sell is:
User sends crypto → directly to an external licensed 
exchange/custodian's address (not AMAX's address)

AMAX should never be the recipient of the inbound crypto. The deposit address provided should belong to the custodian, not AMAX.
Issue 2 — Naming Qin Xiong as Compliance Officer in the UI 🟡
"SELL transactions are processed by AMAX's compliance team (Compliance Officer: Qin Xiong)" — naming a specific individual in a live UI creates personal liability exposure for that person and creates an update obligation every time staff changes. Issue 3 — "Do not send crypto to any address without first receiving written confirmation" 🟡
Good fraud-prevention language — but it creates a process dependency. Issue 4 — Travel Rule obligation not addressed 🔴
Australia has incorporated the Travel Rule through the AML/CTF Amendment Act 2024, with the obligation taking effect from 31 March 2026. All VASPs must collect and transmit originator and beneficiary information for qualifying virtual asset transfers, including self-hosted wallet verification. Zitadelleag
There is no minimum threshold — the Travel Rule applies to all virtual asset transfers regardless of value. Notabene
When a user sends crypto to AMAX's deposit address on a sell, AMAX is the beneficiary VASP and must collect and verify originator information. The current form collects asset type and amount but has no Travel Rule data fields — no originator wallet address capture, no verification of whether the sending wallet is custodial or self-hosted. This is a live compliance gap as of 1 July 2026.
