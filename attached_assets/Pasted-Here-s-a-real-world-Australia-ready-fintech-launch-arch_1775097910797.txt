* Sumsub (best all-in-one)
* Onfido
* Jumio

---

### Flow:

```
User signup
→ email + password
→ MFA setup (TOTP)
→ KYC verification
→ account activated
```

---

### 🔧 Key backend endpoints:

```ts
POST /auth/register
POST /auth/login
POST /auth/mfa/verify
POST /kyc/start
GET  /kyc/status
```

---

### 🚨 Required controls:

* block transactions until KYC = approved
* store KYC status only (NOT raw documents)

## 🔴 LAYER 2 — WALLET + LEDGER (YOUR CORE SYSTEM)

You already built most of this ✅

---

### Must be structured as:

| Component    | Role              |
| ------------ | ----------------- |
| Wallets      | user balances     |
| Ledger       | source of truth   |
| Transactions | immutable history |

---

### 🔧 Required rule:

> ❗ **Ledger = truth, wallet = cache**

---

### Add THIS if not already:

```ts
// every balance change must come from ledger
function applyTransaction(tx) {
  validateTransaction(tx);
  insertLedgerEntry(tx);
  updateWalletBalance(tx);
}
```

---

### 🚨 Critical:

* NO manual balance edits
* ALL flows go through transactions


### Trigger example:

```ts
if (tx.amount > 10000) {
```
