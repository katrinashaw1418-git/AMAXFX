# 🚀 Fintech Launch Checklist (Day 0)

## 🔴 1. Money Movement Safety (MUST PASS)

| Check                   | Pass criteria                                                                 |
| ----------------------- | ----------------------------------------------------------------------------- |
| Idempotency             | Every deposit/transfer API requires `Idempotency-Key` and prevents duplicates |
| Transaction states      | All transactions have `pending → settled/failed/reversed`                     |
| No direct balance edits | All balances derived from ledger (no `wallet +=`)                             |
| Custodian confirmation  | Funds only marked “settled” after custodian confirms                          |
| Retry safety            | API retries cannot double-execute                                             |

✅ If ANY of these fail → **do NOT launch**

---

## 🔴 2. Ledger Integrity (MUST PASS)

| Check                    | Pass criteria                                |
| ------------------------ | -------------------------------------------- |
| Ledger = source of truth | Wallet balances derived from transactions    |
| Reconstruction works     | Historical balances reproducible from ledger |
| Reconciliation exists    | Internal balances vs custodian checked       |
| Drift detection          | Mismatches logged and visible                |
| No silent corrections    | All fixes go through transactions            |

---

## 🔴 3. Custodian Integration (MUST PASS)

| Check                | Pass criteria                         |
| -------------------- | ------------------------------------- |
| Webhooks verified    | Signature validation implemented      |
| Webhook idempotency  | Duplicate webhook handling safe       |
| API failure handling | Failed calls update transaction state |
| Timeouts handled     | No hanging transactions               |
| Mapping consistency  | Custodian tx ↔ your tx 1:1 traceable  |

---

## 🔴 4. Security (MUST PASS)

| Check            | Pass criteria                        |
| ---------------- | ------------------------------------ |
| Authentication   | No hardcoded user IDs                |
| Authorization    | Users can only access their own data |
| Admin protection | MFA for admin actions                |
| Secrets          | API keys NOT in code                 |
| Rate limiting    | Basic abuse protection in place      |

---

## 🔴 5. Data Truthfulness (MUST PASS)

| Check                   | Pass criteria                         |
| ----------------------- | ------------------------------------- |
| No fake valuations      | `manual_nav` not misleading           |
| Missing data visible    | UI shows “unpriced” or “estimated”    |
| Historical source clear | “estimated vs actual” disclosed       |
| No silent NaN/0         | Null handled explicitly               |
| Forecast labeled        | Projection clearly marked as estimate |

---

## 🟠 6. Operations (SHOULD PASS)

| Check              | Pass criteria                       |
| ------------------ | ----------------------------------- |
| Reconciliation job | Runs daily (or more frequently)     |
| Error logging      | Centralized logs (not just console) |
| Alerts             | You know when something breaks      |
| Manual override    | Ability to pause transfers          |
| Backups            | Database backup working             |

| Check            | Pass criteria                    |

| Retry strategy   | Safe retries without duplication |
| Rollback logic   | Failed flows revert cleanly      |
| User messaging   | Errors shown clearly             |

# ⚠️ HARD FAIL CONDITIONS (DO NOT LAUNCH IF TRUE)

* ❌ Duplicate transactions possible
* ❌ Wallet balance ≠ ledger
* ❌ No reconciliation
* ❌ Webhooks not verified
* ❌ Using fake valuations silently
* ❌ Hardcoded user/account access
* ❌ Custodian failures not handled
