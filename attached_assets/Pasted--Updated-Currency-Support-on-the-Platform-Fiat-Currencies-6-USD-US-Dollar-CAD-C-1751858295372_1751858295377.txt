✅ Objective:
Allow users to deposit, hold, transfer, trade, and withdraw USDT and USDC using ERC-20 tokens on Ethereum. These stablecoins will serve as a low-volatility, on-chain alternative to fiat in the dual-rail wealth platform.

🧱 Architecture Additions
🔐 Wallet Layer
Each user gets dedicated ERC-20 wallet addresses for:

USDT (Tether)

USDC (USD Coin)

Generated and managed using:

🔗 Custody provider API (e.g., Fireblocks, BitGo)

OR

🔐 In-house HD wallet service (if custodial)

⚙️ Features by Module
🔸 1. User Wallet Dashboard
Display balances for USDT and USDC (in crypto and fiat terms)

“Deposit” button shows wallet address + QR code

“Withdraw” triggers KYC check and on-chain transaction builder

“Exchange” allows conversion between USDT, USDC, BTC, ETH, and fiat

🔸 2. Trading / FX Engine
New supported trading pairs:

USDT/USDC ↔ BTC/ETH

USDT/USDC ↔ USD/CAD/EUR/GBP

USDT ↔ USDC

Stablecoin quotes via liquidity aggregator or exchange integration

Internal order matching or routed through a partner (VirgoCX)

🔸 3. Deposit Flow
Detect ERC-20 transaction to assigned USDT/USDC address

Confirm on-chain (≥ 1 block)

Credit user's internal stablecoin balance

Log tx hash and value in DB

🔸 4. Withdraw Flow
User inputs destination wallet and amount

KYC status checked

Backend signs ERC-20 transaction and submits it on-chain

Deduct from internal balance, return tx hash

🔸 5. Transaction History
Store metadata:

Token type (USDT/USDC)

Direction (inbound/outbound)

Blockchain tx hash

Fiat equivalent at time of tx

Gas fees (shown in ETH)

🔒 Compliance Considerations
Require full KYC before enabling crypto withdrawals

Monitor stablecoin flows using chain analytics (e.g., TRM Labs or Chainalysis)

Cap transaction size per regulatory jurisdiction

Geo-fence if needed (e.g., USDT restriction in certain countries)
