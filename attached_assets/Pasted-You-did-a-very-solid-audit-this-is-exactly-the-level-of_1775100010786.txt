## 🔴 Gap 1 — No Concurrency Protection (Race Conditions)

Even with DB transactions, you still have:

### Problem

Two simultaneous requests:

```
Request A: withdraw $100
Request B: withdraw $100
Balance = $150
```

Both read → both pass → both deduct → ❌ balance = -$50

---

### ✅ Fix (MANDATORY)

Use **row-level locking**

```ts
const fromWallet = await tx
  .select()
  .from(wallets)
  .where(...)
  .for("update"); // 🔴 CRITICAL
```

👉 Without this, your “atomic” system is still unsafe.

### Problem

