**Real bugs / breakpoints**

1. **Stablecoin deposit and withdrawal are calling a backend route that does not exist**
   In client/src/components/wallets/stablecoin-card.tsx:74-101 and :131-138, both deposit and withdrawal use apiRequest('POST', '/api/transactions', ...).
   But in server/routes.ts, there is a GET /api/transactions route, not a POST /api/transactions route.
   Result: those actions will fail at runtime with a 404 or equivalent API error.

2. **Investment YTD history is hardcoded to 2026**
   In server/routes.ts:1132-1208, the endpoint /api/investments/history-ytd uses:

   * const ANCHOR = new Date("2026-01-01T00:00:00.000Z");
   * returns anchorDate: '2026-01-01'
     This means the chart will become wrong in 2027 and later, and it is already not truly “YTD” in a reusable sense. It should derive the anchor from new Date().getFullYear().

3. **Wallet deposit addresses and QR codes are placeholders, not real operational logic**
   In client/src/pages/wallets.tsx:736-780, the QR code is just a visual mock built with Math.random(), and the addresses are hardcoded sample strings like bc1qxy2... and 0x742d....
   Result: the UI looks functional, but it is not connected to any real wallet generation, custody, or deposit reconciliation flow. That is risky because users could think the addresses are real.

4. **Advisor contact endpoint always says success but does nothing**
   In server/routes.ts:2920-2934, /api/advisor/contact validates the message, then returns success with a comment saying it is only a demo and does not send email or create a ticket.
   Result: from the user’s perspective, messages appear sent even though no delivery happens.

**Important updates required before production**

* **Replace placeholder deposit flow with a real backend path**
  The stablecoin card should post to a real deposit/withdraw endpoint, not /api/transactions, or you need to implement POST /api/transactions properly.

* **Make all year-based charts dynamic**
  The YTD investment endpoint should anchor to the current year, not 2026.

* **Remove fake wallet address UX**
  Do not show copyable blockchain addresses until they are generated from a real wallet/custody system and tied to the authenticated user.

* **Replace demo success endpoints**
  /api/advisor/contact should create a ticket, send an email, or write to a durable queue/log.
