**1) `server/routes.ts`**

* **Issue:** `/api/user` returns `500` for auth failures.
* **Where:** around the `/api/user` handler.
* **Current behavior:** `requireAuth(req)` throws `401`, but the catch block always returns `500`.
* **Patch:**

```ts
app.get("/api/user", async (req, res) => {
  try {
    const auth = requireAuth(req);
    const user = await storage.getUser(auth.userId);
    if (!user) {
      return res.status(404).json({ error: "User not found" });
    }
    const { password: _, ...safeUser } = user;
    return res.json(safeUser);
  } catch (error: any) {
    if (error?.status === 401) {
      return res.status(401).json({ error: error.message || "Unauthorized" });
    }
    return res.status(500).json({ error: "Failed to get user" });
  }
});
```

### P2 — structural but important

**10) `server/routes.ts` / transaction model**

* **Issue:** no immutable double-entry ledger layer.
* **Verdict:** this is a real-money controls gap, not a compile bug.
* **Patch direction:** introduce ledger entries on every completed money movement.

Recommended new table:

```ts
export const ledgerEntries = pgTable("ledger_entries", {
  id: serial("id").primaryKey(),
  userId: integer("user_id").notNull(),
  transactionId: integer("transaction_id").notNull(),
  entryType: text("entry_type").notNull(), // debit | credit
  accountCode: text("account_code").notNull(), // wallet:USD, wallet:BTC, clearing, fees, etc
  currency: text("currency").notNull(),
  amount: decimal("amount", { precision: 20, scale: 8 }).notNull(),
  createdAt: timestamp("created_at").defaultNow().notNull(),
});
```

And for each completed transfer/exchange/investment:

* write one debit entry
* write one credit entry
* optionally a fee entry

This is not the first patch to do, but it is the patch that moves the system toward auditability.
