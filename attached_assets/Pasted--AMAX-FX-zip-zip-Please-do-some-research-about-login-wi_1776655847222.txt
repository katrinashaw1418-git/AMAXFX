Step 5 — Add the backend endpoint to server/routes.ts
Install the Google auth library on the server side:

Then add this route in server/routes.ts (near your other /api/auth/ routes):

typescript
import { OAuth2Client } from 'google-auth-library';

const googleClient = new OAuth2Client(process.env.GOOGLE_CLIENT_ID);

// POST /api/auth/google
app.post('/api/auth/google', async (req, res) => {
  const { access_token } = req.body;
  if (!access_token) return res.status(400).json({ error: 'Missing access_token' });

  try {
    // Fetch profile from Google using the access token
    const googleRes = await fetch(
      `https://www.googleapis.com/oauth2/v2/userinfo?access_token=${access_token}`
    );
    if (!googleRes.ok) throw new Error('Failed to fetch Google profile');
    
    const profile = await googleRes.json();
    const { email, given_name, family_name, id: googleId } = profile;

    // Find existing user or create a new one
    let user = await storage.getUserByEmail(email);
    
    if (!user) {
      // New user — create account (no password needed for OAuth users)
      user = await storage.createUser({
        email,
        firstName: given_name || '',
        lastName: family_name || '',
        username: email,
        passwordHash: null, // OAuth users have no password
        googleId,
        emailVerified: true, // Google already verified their email
      });
    } else if (!user.googleId) {
      // Existing email account — link Google ID to it
      await storage.updateUser(user.id, { googleId, emailVerified: true });
    }

    const token = signToken({ userId: user.id, username: user.email, email: user.email });
    res.json({ token, user: { id: user.id, email: user.email, firstName: user.firstName } });
    
  } catch (err: any) {
    console.error('Google auth error:', err);
    res.status(401).json({ error: 'Google authentication failed' });
  }
});

Mobile / Phone sign-up
For phone number login, the standard approach for a fintech app is Firebase Authentication (for the SMS OTP flow) or Twilio Verify. Given you already have email OTP infrastructure (/api/auth/verify-otp), the easiest path is:

Add a phone input field (with country code selector using react-phone-number-input)
Hit a new /api/auth/send-phone-otp endpoint that calls Twilio Verify API
User enters the 6-digit code — same OTP UI you already have
Verify against Twilio, then create/find the user
Given AUSTRAC compliance requirements, phone verification is actually good to have as it adds a second factor of identity.

Apple Sign-In
Apple Sign-In requires an Apple Developer account ($99/year), a Service ID, and a private key. The flow is more complex. Recommend implementing Google first, then Apple when you have the Apple Developer enrollment sorted. The library react-apple-signin-auth handles the frontend side.
